Protégé MCP Server

An MCP (Model Context Protocol) server for Protégé Desktop, packaged as a single OSGi plugin. It exposes the live, active ontology of a running Protégé to MCP clients (Claude Code / Desktop, Codex CLI, VS Code MCP, ...) over a localhost HTTP endpoint. Reads and edits flow through Protégé's shared OWLModelManager, so they appear in the GUI immediately and join the undo stack - exactly like manual edits.

Requirements

Protégé must run on a Java 17+ JVM: the MCP Java SDK and the embedded Jetty 12 host are Java 17 bytecode, and the OSGi manifest declares Require-Capability: osgi.ee ... JavaSE 17. Select a 17+ JDK with the PROTEGE_JAVA_HOME environment variable before launching Protégé.

What you get

  • A MCP Server view (start / stop, the bound URL and bearer token) and a Settings > MCP tab.
  • 26 tools across explore / search, edit, history, and reasoning - every edit goes through OWLModelManager.applyChanges, so it appears in the GUI and joins the shared undo stack.
  • Loopback-only HTTP with OAuth 2.1 (dynamic registration + browser consent, persisted across restarts) and a static bearer token, supported in parallel.

Version information

v0.1.1

  • Complete OWL 2 incremental authoring surface: add_axiom now covers 38 axiom types (property characteristics, chains, inverses, n-ary class/property/individual axioms, has-key, negative assertions, declarations, datatype definitions, annotation-property axioms), every axiom takes an optional annotations operand (reified owl:Axiom), and add_annotation supports typed / IRI-valued annotation values.
  • New ontology-header tools: set_ontology_id, add_import / remove_import, and add_ontology_annotation / remove_ontology_annotation. Data ranges accept Manchester syntax (e.g. xsd:integer[>= 0], {1, 2, 3}).
  • merge_ontology_document for lossless bulk import, plus robustness fixes (ontology-id collision guard, longer merge timeout, Windows path handling, unresolved-import warning).

v0.1.0

  • First release - the in-Protégé MCP server (Architecture Approach A).
  • Embedded MCP SDK 2.0.0 + Jetty 12 (Jakarta Servlet 6) inlined into the bundle; Protégé / OWLAPI / Guava / SLF4J stay provided.
