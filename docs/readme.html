Protégé MCP Server

An MCP (Model Context Protocol) server for Protégé Desktop, packaged as a single OSGi plugin. It exposes the live, active ontology of a running Protégé to MCP clients (Claude Code / Desktop, Codex CLI, VS Code MCP, ...) over a localhost HTTP endpoint. Reads and edits flow through Protégé's shared OWLModelManager, so they appear in the GUI immediately and join the undo stack - exactly like manual edits.

Requirements

Protégé must run on a Java 17+ JVM: the MCP Java SDK and the embedded Jetty 12 host are Java 17 bytecode, and the OSGi manifest declares Require-Capability: osgi.ee ... JavaSE 17. Select a 17+ JDK with the PROTEGE_JAVA_HOME environment variable before launching Protégé.

What you get

  • A MCP Server view (start / stop, the bound URL and bearer token) and a Settings > MCP tab.
  • 33 tools across explore / search, edit, documents, history, and reasoning - every model edit goes through OWLModelManager.applyChanges, so it appears in the GUI and joins the shared undo stack (document open/save use Protégé's own load/save APIs).
  • Loopback-only HTTP with OAuth 2.1 (dynamic registration + browser consent, persisted across restarts) and a static bearer token, supported in parallel.

Version information

v0.2.1

  • Tool-driven construction ergonomics (37 → 41 tools). set_active_ontology switches which loaded ontology your edits target; load_ontology (keep_active) and add_import (document=…) resolve imports without stealing the active ontology.
  • apply_changes applies a previewed batch of axiom add/remove operations in one call and one undo entry, reporting per-operation results and any new entities introduced; strict=true refuses to mint a brand-new entity from an unrecognized IRI/name (a signal now surfaced by every write tool).
  • set_label upserts an rdfs:label; set_prefix registers a prefix; create_class / create_entity gain namespace / label / label_lang / no_label for term-namespace and language-tagged labels.
  • Manchester <IRI> operands resolve inside compound class expressions; validate_ontology (with_reasoner=true) adds a consistency / unsatisfiable-class verdict; get_entity_context neighbours are structured {iri, display, type}.

v0.2.0

  • Natural-language layer: every tool now returns a structured JSON object (mirrored as text), so an LLM client gets machine-readable results instead of prose.
  • Orientation & safety tools: get_ontology_context / get_entity_context summarise the ontology or a single term's neighbourhood; preview_changes shows an edit's diff (and any new entities) before it is applied; validate_ontology audits modelling quality (missing labels/definitions, duplicate labels, undeclared entities, properties with no domain/range, subclass cycles, …).
  • Guided workflows as MCP prompts: audit_ontology, explain_class, add_subclass_safely, find_and_fix_unsatisfiable, model_domain.
  • validate_ontology is import-aware: the per-entity quality checks audit only the terms the active ontology is responsible for, so imported BFO/IOF terms are not false-flagged for label/definition/domain/range that lives upstream (set include_imports=true to audit the whole imports closure).

v0.1.2

  • load_ontology opens a local path or URL into the workspace as its own active ontology and makes it active. The document is fetched off the UI thread (so a slow remote fetch never freezes Protégé) and wired in with Protégé's own copy-ontology / activate path — no modal load dialogs; not undoable.
  • Higher-level editing tools: rename_entity / delete_entity rewrite or remove an entity (and its referencing axioms) across the active ontology, undoable.
  • Reasoning & query tools: list_reasoners / set_reasoner to choose the reasoner; execute_dl_query (Manchester class expression → equivalent / sub / super / instances, the DL Query workbench); and get_explanations for minimal justifications behind an entailment.

v0.1.1

  • Complete OWL 2 incremental authoring surface: add_axiom now covers 38 axiom types (property characteristics, chains, inverses, n-ary class/property/individual axioms, has-key, negative assertions, declarations, datatype definitions, annotation-property axioms), every axiom takes an optional annotations operand (reified owl:Axiom), and add_annotation supports typed / IRI-valued annotation values.
  • New ontology-header tools: set_ontology_id, add_import / remove_import, and add_ontology_annotation / remove_ontology_annotation. Data ranges accept Manchester syntax (e.g. xsd:integer[>= 0], {1, 2, 3}).
  • merge_ontology_document for lossless bulk import, plus robustness fixes (ontology-id collision guard, longer merge timeout, Windows path handling, unresolved-import warning).

v0.1.0

  • First release - the in-Protégé MCP server (Architecture Approach A).
  • Embedded MCP SDK 2.0.0 + Jetty 12 (Jakarta Servlet 6) inlined into the bundle; Protégé / OWLAPI / Guava / SLF4J stay provided.
